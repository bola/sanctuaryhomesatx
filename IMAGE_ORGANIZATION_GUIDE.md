## Image Naming Convention

Use descriptive, lowercase names with hyphens:

### Pattern
```
[project-name]-[space/view]-[number].jpg
```

### Examples
```
sylvan-drive-exterior-front-01.jpg
sylvan-drive-living-room-01.jpg
sylvan-drive-kitchen-02.jpg
bouldin-avenue-pool-01.jpg
kinney-avenue-master-bedroom-01.jpg
```

### Space/View Types
- `exterior-front`
- `exterior-rear`
- `exterior-side`
- `curb-appeal`
- `living-room`
- `dining-room`
- `kitchen`
- `master-bedroom`
- `bedroom`
- `bathroom`
- `master-bath`
- `outdoor-living`
- `pool`
- `landscape`
- `detail`
- `architectural-detail`

## Image Requirements Per Project

### Minimum Images
- **1 Hero Image** - Best overall shot of the project
- **2-3 Exterior Views** - Different angles/perspectives
- **4-6 Interior Views** - Key living spaces
- **1-2 Detail Shots** - Craftsmanship close-ups

### Ideal Image Count
- **8-12 images per project** for rich galleries
- Mix of wide shots and detail shots
- Include both day and dusk/night shots if available

---

## Image Technical Specifications

### Resolution
- **Minimum:** 1800px on longest side
- **Preferred:** 2400px on longest side for hero images
- **Aspect Ratio:** Preferably 3:2 or 4:3 for consistency

### Format
- **Original:** Keep as high-quality JPEG
- **Web Optimized:** We'll create WebP versions during development
- **Quality:** 80-90% JPEG quality

### File Size
- **Original:** Keep originals (even if 2-5MB each)
- **Optimized versions** will be created automatically during build

## Next Steps After Images Are Ready

Once all images are downloaded and organized:

1. **Optimize Images**
   - Create WebP versions
   - Generate multiple sizes for responsive design
   - Compress without losing quality

2. **Build Image Data**
   - Create JSON file with image metadata
   - Include captions, alt text, credits

3. **Implement Galleries**
   - Build lightbox functionality
   - Add carousel for hero images
   - Implement lazy loading

4. **Test Display**
   - Verify all images load correctly
   - Check responsive behavior
   - Test on multiple devices
